# 更新日志

这里记录 QQ 空间 Ultra 面向用户的重要变化。

## [0.3.1] - 2026-05-23

### 说说卡片显示修复

- 修复查看图文说说时，卡片可能只显示文字、不显示图片的问题。
- 修复部分说说卡片时间显示为当前时间，而不是原说说发布时间的问题。
- 当 QQ 空间返回的数据里没有可靠发布时间时，卡片会显示“未知时间”，不再用当前时间误导判断。
- 优化 QQ 空间多种图片字段和时间字段的识别，提升看说说、读说说、评论说说、点赞说说和详情卡片的一致性。

### 稳定性与安全性

- 详情接口缺少图片信息时，会尝试复用列表中已经读取到的图片，减少图文说说展示不完整的情况。
- 收紧说说卡片图片来源识别，只使用明确的远程图片地址，避免异常数据被误当作图片加载。

## [0.3.0] - 2026-05-23

### 自动绑定更省心

- 插件安装、下载后重载或 AstrBot 启动后，会自动尝试绑定 QQ 空间。
- 如果 OneBot / aiocqhttp 客户端稍后才连接，插件会在检测到后继续尝试自动绑定。
- 自动绑定遇到临时失败时会最多重试 3 次，减少刚启动、平台未完全就绪或网络短暂波动带来的失败。
- 已经绑定成功或已有可用登录态时，插件会直接复用，不会重复绑定。

### 首次使用体验优化

- 大多数 OneBot v11 / aiocqhttp 环境下，首次使用前不再需要手动执行 `/qzone autobind`。
- 如果平台无法提供 Cookie，仍然可以使用 `/qzone autobind` 或 `/qzone bind <cookie>` 手动绑定。
- 自动绑定不会拖慢插件启动、重载或 AstrBot 加载流程。
- 已有 Cookie 时，QQ 空间服务会提前准备好，首次查看、发布或互动会更顺滑。

### 稳定性改进

- 修复首次自动绑定失败后，后续 OneBot 连接就绪却不再自动重试的问题。
- 修复忽略群或忽略用户的消息被跳过时，自动绑定也被一并跳过的问题。
- 优化自动读空间和自动绑定同时发生时的处理，避免重复绑定或重复等待。
- 优化插件停用和重载时的清理流程，减少异常。

### 安全说明

- 自动绑定仍然只使用当前 AstrBot / OneBot 环境提供的能力。
- 插件不会内置默认 Cookie，也不会写入任何固定登录凭据。
- 手动绑定相关命令仍然保持管理员权限限制。
- Cookie、密钥等敏感信息不会出现在配置说明或更新日志中。

## 历史能力概览

- 支持看说说、读说说、评论、点赞、发布、AI 写说说、删除、回评等 QQ 空间操作。
- 支持投稿、匿名投稿、撤稿、查看稿件、通过稿件、拒绝稿件等投稿审核流程。
- 支持自动发说说、自动评论、评论后点赞和管理员反馈。
- 支持 OneBot v11 / aiocqhttp 自动 Cookie 获取，也支持手动 Cookie 绑定。
- 支持独立服务处理 QQ 空间请求、Cookie 管理、图片处理和结果渲染。
- 支持说说卡片、评论卡片和发布结果渲染。
- 支持 AI 工具，让管理员可以通过模型完成查看、发布、评论、点赞和删除等操作。
